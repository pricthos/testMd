# VibeX 유저 매뉴얼 (User Manual)
> **최종 수정일:** 2026년 3월 3일  
> AI와 함께 비즈니스의 '바이브'를 코드로 바꾸는 VibeX 플랫폼 가이드입니다.

---

## 1. 시작하기 (Getting Started) {#getting-started}

### 1.1 회원가입 및 로그인 {#signup-login}
VibeX는 사용자의 편의를 위해 일반 이메일 가입과 Google 소셜 로그인 방식을 모두 지원합니다.

* **이메일 가입:** 로그인 화면 하단의 '회원가입'을 눌러 진행할 수 있습니다. 필수 약관 동의 후 발송되는 이메일 인증을 거치면 완료됩니다. 인증 메일이 보이지 않는다면 스팸 메일함을 확인해 주세요.
* **Google 소셜 로그인:** 'Google로 로그인' 버튼을 클릭하여 간편하게 연동할 수 있습니다. 팝업이 뜨지 않는다면 브라우저의 팝업 차단 설정을 확인해 주세요.

> 💡 **접속 문제가 발생했나요?** > 가입 버튼이 반응하지 않거나 무한 로딩이 발생할 경우, 광고 차단(AdBlock) 확장 프로그램을 잠시 끄시거나 시크릿 모드에서 접속해 보시는 것을 권장합니다.

### 1.2 계정 및 보안 설정 {#account-settings}
* **비밀번호 재설정:** 이메일 계정은 로그인 화면의 '비밀번호를 잊으셨나요?' 링크를 통해 재설정 가능합니다. Google 사용자는 Google 계정 설정에서 직접 변경해야 합니다.
* **계정 탈퇴 및 데이터 파기:** `[설정] > [계정 관리]`에서 진행 가능하며, 탈퇴 시 모든 데이터는 영구 삭제되어 복구가 불가능합니다. 중요한 코드는 반드시 미리 **내보내기(Export)** 하세요.

### 1.3 요금제 및 X 안내 {#pricing-and-x}
**X**란 VibeX 내에서 AI 작업(앱 기획, 코드 생성/수정) 시 소모되는 전용 재화입니다. 연간 구독 시 **약 20%의 할인** 혜택을 제공합니다.

## 2. 프로젝트 관리 및 대시보드 {#dashboard}

### 2.1 대시보드 개요 및 탐색 {#dashboard-overview}
* **중단한 곳에서 계속하기:** 메인 홈 중앙에서 최근 작업하던 프로젝트로 즉시 복귀할 수 있습니다.
* **앱 목록 및 검색:** 상단 검색바와 워크스페이스 드롭다운 필터를 통해 프로젝트를 빠르게 찾을 수 있습니다.

### 2.2 앱(프로젝트) 생성 및 관리 {#manage-projects}
* **생성:** 우측 상단 `+ 새 앱` 버튼을 클릭합니다.
* **수정/복제:** 프로젝트 카드의 더보기(⋮) 메뉴에서 이름 변경, 복제, 삭제가 가능합니다. 
* ⚠️ **주의:** 삭제된 프로젝트는 절대 복구할 수 없습니다.

### 2.3 팀 워크스페이스 {#team-workspace}
* **팀원 초대:** 설정에서 이메일로 동료를 초대하고 권한(편집자, 뷰어)을 부여할 수 있습니다. 현재 프로젝트 소유권(Owner) 양도는 지원하지 않습니다.

## 3. AI 기반 앱 기획 및 생성 {#app-planning}

### 3.1 대화형 앱 기획 {#conversational-planning}
* **프롬프트 입력:** 아이디어를 텍스트로 입력하고 `Enter`를 누르세요. 구체적인 타겟과 컬러를 명시할수록 퀄리티가 높아집니다.
* **플랫폼 지정:** 기본은 웹 기반입니다. 모바일 환경을 원하시면 "Mobile-first UI"를 명시해 주세요.

### 3.2 기획안 고도화 {#refining-plan}
"페이지 리스트를 표로 정리해 줘", "DB 스키마를 정의해 줘" 등 구체적인 개발 명세를 요구하여 기획 완성도를 높일 수 있습니다.

### 3.3 프로젝트 생성 파이프라인 {#generation-pipeline}
`프로젝트 생성` 버튼 클릭 시 **Starting → Setting up → Thinking → Generating** 단계를 거칩니다. 에러 감지 시 자동 복구(Auto-fix) 로직이 실행되어 진행 바가 초기화될 수 있습니다.

---

## 4. 에디터 및 실시간 미리보기 {#editor-preview}

### 4.1 에디터 인터페이스 {#editor-interface}
* **코드 탭:** 프론트엔드 소스 코드 직접 편집.
* **데이터 탭:** 샘플 JSON 및 DB 구조 관리.
* **대시보드 탭:** 배포 상태 및 환경 변수 설정.

### 4.2 실시간 미리보기 및 로그 {#preview-logs}
우측 영역에서 실시간 렌더링을 확인하고, 하단 `콘솔` 버튼을 통해 자바스크립트 로그를 디버깅할 수 있습니다.

### 4.3 코드 수정 및 버전 관리 {#code-versioning}
* **자동 저장:** 실시간으로 반영되나 수동 편집 직후 새로고침은 피하세요.
* **되돌리기:** 이전 결과가 좋다면 히스토리 패널에서 과거 시점으로 롤백할 수 있습니다.

---

## 5. 프롬프트 작성 가이드 {#prompting-tips}

### 5.1 디자인 및 UI 최적화 {#design-optimization}
* "Linear 스타일", "애플 스타일" 등 구체적 레퍼런스를 제시하세요.
* `#1E1E1E` 같은 HEX 코드나 `padding: 24px` 같은 구체적 수치를 활용하세요.

### 5.2 복잡한 로직 구현 {#logic-implementation}
* "isLoggedIn 상태 관리 로직 포함", "Stripe 결제 모달 연동 UI 흐름 작성" 등 사용자 시나리오를 구체적으로 지시하세요.

### 5.3 효과적인 에러 해결 {#troubleshooting}
* 콘솔의 빨간색 에러 로그를 복사해서 챗봇에게 "이 에러를 고쳐줘"라고 요청하세요.
* 수정 시 "기존 레이아웃은 100% 유지해 줘"라고 명시하여 디자인 틀어짐을 방지하세요.

## 6. 커뮤니티 템플릿 {#community-templates}

### 6.1 템플릿 탐색 및 가져오기 {#import-templates}
다른 유저의 프로젝트를 내 워크스페이스로 `가져오기(Import)` 할 수 있습니다. 가져온 후에는 본인의 X를 사용하여 자유롭게 수정이 가능합니다.

### 6.2 라이선스 주의사항 {#license-notes}
템플릿 내 이미지는 저작권 문제가 있을 수 있으므로 상업적 사용 시 본인의 에셋으로 교체하는 것을 권장합니다.

---

## 7. 백엔드 및 고급 연동 {#backend-integrations}

### 7.1 백엔드 기능(Functions) 활용 {#backend-functions}
MongoDB, Supabase 등 외부 DB 연결 및 외부 API 호출이 가능합니다 (Ultra 플랜 이상).

### 7.2 환경 변수(.env) 설정 {#env-variables}
API Secret Key 등 민감한 정보는 반드시 **대시보드 > 환경 변수** 메뉴에 저장하여 보안을 유지하세요.

### 7.3 GitHub 연동 {#github-integration}
프로젝트 소스 코드를 내 저장소로 `Push` 할 수 있습니다. 빈 Repository를 연결하는 것이 가장 안정적입니다.

---

## 8. 배포 및 내보내기 {#publish-export}

### 8.1 앱 배포하기 {#app-publishing}
`게시(Publish)` 버튼 클릭 시 즉시 라이브 URL이 발급됩니다. 수정 후에는 다시 게시 버튼을 눌러야 반영됩니다.

### 8.2 커스텀 도메인 연결 {#custom-domain}
도메인 구매처의 DNS 설정에 VibeX가 제공하는 레코드 값을 등록하세요. SSL 인증서는 자동으로 발급됩니다.

### 8.3 소스 코드 다운로드 {#download-source}
전체 프로젝트를 **ZIP 파일**로 내려받아 로컬 환경(`npm install` → `npm run dev`)에서 실행하거나 외부 호스팅에 배포할 수 있습니다.
